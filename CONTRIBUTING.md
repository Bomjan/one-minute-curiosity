# Laboratory Notebook Standards

To maintain the "WOW" factor for visitors, every new episode added to this repository MUST follow the **One-Minute Curiosity Blueprint**.

## Folder Structure
Every curiosity gets its own folder named: `episode-[XX]-[slug]`
Example: `episode-04-fibonacci-chaos`

## File Requirements

### 1. `code.py`
- **Minimalist**: Only the essential logic shown in the video.
- **Humanized Comments**: Explain the *intent*, not just the syntax.
- **Top-Level Entry**: Must be runnable without complex setups.

### 2. `explanation.md`
- **The Hook**: Start with a 1-sentence summary of the curiosity.
- **The Logic**: Break down the math or algorithm in simple terms.
- **The Result**: What did we learn?

### 3. `visual.png` (Optional)
- A high-res screenshot or diagram that helps explain the concept at a glance.

## Writing Style
- Tone: Curious, professional, and encouraging.
- Format: Clean Markdown with proper headers.
